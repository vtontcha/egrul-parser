# egrul-parser

Сборщик карточек организаций из ЕГРЮЛ через api-fns.ru.
Сохраняет данные в JSON.

## Почему api-fns.ru, а не egrul.nalog.ru

Официальный сайт egrul.nalog.ru не имеет открытого API — только веб-форму с капчей.
api-fns.ru — сторонний сервис-агрегатор, который работает с базой ФНС и предоставляет
удобный JSON API. Бесплатный ключ выдаётся сразу после регистрации на сайте.

## Руководство пользователя

## Получите API-ключ

1. Зайдите на [api-fns.ru](https://api-fns.ru) и зарегистрируйтесь.
2. После входа перейдите в **«Мои тарифы»** и подключите бесплатный тариф **«СПК-старт»**.
3. В личном кабинете скопируйте ваш **40-значный ключ** — его нужно прописать в .env в строку API_FNS_KEY=ваш_40_значный_ключ_здесь.

> Бесплатный тариф включает ограниченное число запросов в сутки.
> Для сбора данных с нескольких страниц (по ~10 компаний на страницу) его вполне достаточно.

### Установка

```bash
# 1. Клонировать репозиторий
git clone <ваш-репозиторий>
cd egrul-parser

# 2. Создать виртуальное окружение
python -m venv .venv          # Windows
python3 -m venv .venv         # macOS / Linux

# 3. Активировать
.venv\Scripts\activate        # Windows
source .venv/bin/activate     # macOS / Linux

# 4. Установить зависимости
pip install -r requirements.txt

# 5. Создать файл с ключом
echo API_FNS_KEY=ваш_ключ > .env
```

### Запуск

```bash
python main.py
```

### Изменить число страниц

Откройте `config.py`:

```python
PAGES_TO_PARSE: int = 10   # собрать 10 страниц (~100 компаний)
```

### Изменить выборку

```python
# Только IT-компании (ОКВЭД 62)
SEARCH_QUERY: str = "any"
SEARCH_FILTER: str = "okved=62"

# Только Москва
SEARCH_FILTER: str = "region=77"

# Поиск по названию
SEARCH_QUERY: str = "Газпром"
SEARCH_FILTER: str = ""
```

### Возможные проблемы

| Симптом | Причина | Решение |
|---|---|---|
| `{"error": "wrong key"}` | Неверный или не подключённый ключ | Проверьте `.env`, убедитесь что тариф активирован на api-fns.ru |
| Поля карточки пустые | Имена ключей в JSON не совпадают | Добавьте `print(raw)` в `parser.py`, посмотрите реальные ключи |
| `HTTP 429` | Превышен лимит запросов тарифа | Увеличьте `REQUEST_DELAY` в `config.py` или дождитесь сброса лимита |
| `Таймаут` | Временная недоступность сервиса | Скрипт сам повторит попытку 3 раза; если не помогло — подождите и запустите снова |
